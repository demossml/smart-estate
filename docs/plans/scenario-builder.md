# Конструктор сценариев — план

> **Для Hermes:** реализовать по задачам, после утверждения плана Гришей.

**Цель:** заменить текущую форму создания сценария (1 триггер + 1 действие) на гибкий конструктор, где можно собирать цепочки из нескольких условий и действий — как из кубиков.

**Принцип:** backend уже всё умеет (много условий, много действий, ANY/ALL). Нужен только новый фронтенд.

---

## Пример из жизни (use case Гриши)

```
Подъехала машина → BLE/WiFi опознал →
  ├─ Открыть ворота
  ├─ Открыть гараж
  └─ Включить свет в гараже

Датчик движения в коридоре →
  └─ Включить свет в коридоре
```

Один сценарий = несколько условий + несколько действий. Можно комбинировать датчики из разных комнат.

---

## Концепт интерфейса

```
┌─────────────────────────────────────┐
│  НАЗВАНИЕ: [Вечерний режим      ]  │
├─────────────────────────────────────┤
│                                     │
│  ЕСЛИ  (логика: ○ ЛЮБОЕ  ● ВСЕ)   │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 🌡 Датчик гостиной         ▸│   │
│  │    Температура < 20°C       │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │ 🕐 Время                    ▸│   │
│  │    После заката             │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │ 👤 Датчик движения         ▸│   │
│  │    Движение = Есть          │   │
│  └─────────────────────────────┘   │
│                                     │
│  [+ Добавить условие]              │
│                                     │
│  ─────────────────────────────────  │
│                                     │
│  ТО                                │
│                                     │
│  ┌─────────────────────────────┐   │
│  │ 💡 Свет гостиной    [ВКЛ]  │   │
│  │    ─── или ───               │   │
│  │    Яркость: [═════●════] 70% │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │ 🔒 Ворота           [ОТКР] │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │ ⏱ Пауза 2 мин              │   │
│  └─────────────────────────────┘   │
│  ┌─────────────────────────────┐   │
│  │ 💡 Свет гостиной    [ВЫКЛ] │   │
│  └─────────────────────────────┘   │
│                                     │
│  [+ Добавить действие]             │
│                                     │
├─────────────────────────────────────┤
│  [Сохранить]          [Отмена]     │
└─────────────────────────────────────┘
```

Каждый блок (условие или действие) — отдельная «карточка». Можно добавлять сколько угодно. Порядок действий важен (выполняются сверху вниз).

## Типы условий

| Тип | Что выбирает пользователь | Пример |
|-----|--------------------------|--------|
| **Датчик** | Устройство → свойство → оператор → значение | `Гостиная t° < 20` |
| **Время** | Тип (закат/рассвет/время/cron) → смещение | `Закат − 30 мин` |
| **Состояние** | Устройство → состояние | `Ворота = Открыты` |

## Типы действий

| Тип | Что выбирает | Пример |
|-----|-------------|--------|
| **Устройство** | Какое → команда (ВКЛ/ВЫКЛ/ОТКР/ЗАКР) | `Свет кухни ВКЛ` |
| **Группа** | Комната → все устройства типа X → команда | `Все лампы в Гостиной ВЫКЛ` |
| **Пауза** | Длительность (сек/мин) | `Ждать 2 мин` |
| **Сценарий** | Другой сценарий → включить/выключить | `Активировать «Ночной режим»` |

## Задачи

### 1. Создать типы TypeScript для конструктора
- Файл: `client-app/src/types/scenario-builder.ts`
- Типы: `Condition`, `Action`, `BuilderState`, `ConditionType`, `ActionType`
- `Condition` — дискриминируемый union: `{type:'device', device, property, operator, value}` | `{type:'time', kind:'sunset'|'sunrise'|'cron', offset?}` | `{type:'state', device, expectedState}`
- `Action` — `{type:'device', device, command}` | `{type:'group', room, deviceType, command}` | `{type:'delay', seconds}`

### 2. Утилиты сборки JSON для API
- Файл: `client-app/src/lib/scenario-codec.ts`
- `buildTriggers(conditions: Condition[], logic: 'ANY'|'ALL'): string` → `triggers_json`
- `buildActions(actions: Action[]): string` → `actions_json`
- `parseTriggers(json: string): Condition[]` — для редактирования
- `parseActions(json: string): Action[]`

### 3. Компонент ConditionCard
- Файл: `client-app/src/components/scenario/ConditionCard.tsx`
- Пропсы: `condition`, `onChange`, `onDelete`, `devices`
- Три режима в зависимости от `condition.type`:
  - **device**: select датчика → select свойства → select оператора → input значения
  - **time**: select вида (закат/рассвет/время) → input минут смещения
  - **state**: select устройства → select состояния (открыто/закрыто/вкл/выкл)

### 4. Компонент ActionCard
- Файл: `client-app/src/components/scenario/ActionCard.tsx`
- Пропсы: `action`, `onChange`, `onDelete`, `devices`, `rooms`
- Режимы:
  - **device**: select устройства → select команды
  - **group**: select комнаты → select типа устройств → select команды
  - **delay**: input секунд/минут

### 5. Переписать Scenarios.tsx с конструктором
- Заменить текущий builder на список `ConditionCard` + `ActionCard`
- Кнопки «+ Добавить условие», «+ Добавить действие»
- Переключатель логики ANY / ALL (radio buttons)
- Превью в виде читаемого текста
- Сохранение через `api.createScenario()`
- Редактирование существующего (загрузка через `parseTriggers`/`parseActions`)

### 6. Бекенд: группы устройств (опционально, фаза 2)
- `POST /api/groups/:id/all-on` / `all-off` уже есть
- Нужно: endpoint действия «комната → все устройства типа X»
- Или сделать на фронте: развернуть группу в список отдельных device-actions
