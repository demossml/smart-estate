## 1. Платформы и устройства

### 1.1 Поддерживаемые платформы

| Платформа | Версия | Приоритет |
|---|---|---|
| iOS Safari | 15.0+ | **P0 — основной** |
| Android Chrome | 95+ | **P0 — основной** |
| Desktop Chrome/Safari | последняя | P1 — важно |
| Desktop Firefox/Edge | последняя | P2 — опционально |

### 1.2 Режимы работы

Приложение — Progressive Web App (PWA):
- Открывается по URL в любом браузере
- Можно установить на домашний экран (iOS: «На экран Домой», Android: «Установить»)
- После установки работает как нативное: без адресной строки, свой Splash Screen
- Service Worker кэширует статику → работает офлайн (данные закэшированы)
- WebSocket переподключается при сворачивании/разворачивании

### 1.3 Целевые разрешения (Mobile First)

| Устройство | Ширина | Высота (видимая)* | Статус |
|---|---|---|---|
| iPhone SE (2020) | 375pt | 548pt | **Базовый макет** |
| iPhone 14 / 15 | 390pt | 632pt | Основной |
| iPhone 14 Pro Max | 430pt | 718pt | Максимальный |
| Android Compact | 360pt | 520pt | Минимальный |
| Android Standard | 412pt | 680pt | Основной |
| **iPad Mini** | **744pt** | **880pt** | **Планшет компактный** |
| **iPad Air/Pro 11"** | **834pt** | **1000pt** | **Планшет основной** |
| **iPad Pro 12.9"** | **1024pt** | **1250pt** | **Планшет большой** |
| **Android Tablet 10"** | **800pt** | **1050pt** | **Планшет Android** |

> \* Видимая высота = экран - Status Bar - Header (44pt) - Tab Bar (56pt) ≈ минус 100pt

### 1.4 Breakpoints (адаптивная сетка)

```
Ширина < 430pt    → 1 колонка (телефон)
Ширина 430-767pt  → 2 колонки (большой телефон / компактный планшет)
Ширина 768-1023pt → 3 колонки (планшет)
Ширина ≥ 1024pt   → до 4 колонок + боковая панель (большой планшет / десктоп)
```

### 1.5 Планшетный режим (Sidebar Layout)

На планшетах (≥ 768pt) используется двухпанельный макет:

```

│   SIDEBAR    │        КОНТЕНТ               │
│   280pt      │                              │

│ 💡 Устройства│                              │
│ 🌡️ Климат   │                              │
│ 🚪 Ворота    │                              │
│ ⏱️ Сценарии │                              │
```

- Sidebar всегда виден, текущий экран выделен accent-blue
- Контентная область занимает оставшуюся ширину
- Навигация мгновенная (без transition, как в iPad Settings)

## 2. Сетка и отступы

### 2.1 Базовая сетка

```
┌──────────────────────────────────────┐
│  ← 16pt отступ слева                 │
│                                      │
│  ┌──────────────────────────────┐   │
│  │  Контентная область           │   │
│  │  Ширина = экран - 32pt       │   │
│  └──────────────────────────────┘   │
│                                      │
│                    16pt отступ →     │
└──────────────────────────────────────┘
```

### 2.2 Система отступов (дизайн-токены)

| Токен | Значение | Где применяется |
|---|---|---|
| `space-xs` | 4pt | Иконка ↔ текст внутри pill |
| `space-sm` | 8pt | Между элементами в строке |
| `space-md` | 12pt | Между плитками устройств |
| `space-lg` | 16pt | Внешние отступы экрана, между карточками |
| `space-xl` | 24pt | Между секциями экрана |
| `space-2xl` | 32pt | Заголовок ↔ контент |

### 2.3 Радиусы скругления

| Элемент | Радиус |
|---|---|
| Кнопки (Button) | 12pt |
| Карточки (Card) | 16pt |
| Плитки устройств (Device Tile) | 12pt |
| Модальные окна | 20pt (верхние углы) |
| Инпуты / поля ввода | 10pt |
| Toggle (переключатель) | 16pt |

## 3. Цвета и типографика

### 3.1 Цветовая схема (Тёмная тема, по умолчанию)

```
ФОН (Background)
├── bg-primary:      #0D1117     → фон экрана
├── bg-secondary:    #161B22     → фон карточек
├── bg-tertiary:     #21262D     → фон инпутов, выделенных элементов

ТЕКСТ (Text)
├── text-primary:    #E6EDF3     → основной текст
├── text-secondary:  #8B949E     → подписи, описания
├── text-tertiary:   #484F58     → placeholder, disabled

АКЦЕНТЫ (Accent)
├── accent-blue:     #00B4FF     → информация, ссылки
├── accent-green:    #00FF9D     → вкл / ок / успех
├── accent-orange:   #FF8C00     → предупреждение
├── accent-red:      #FF3860     → ошибка / тревога
├── accent-yellow:   #FFD700     → режим ожидания
├── accent-purple:   #B060FF     → AI / сценарии

ГРАНИЦЫ (Border)
├── border-default:  rgba(255,255,255,0.06) → границы карточек
├── border-active:   rgba(0,180,255,0.25)   → активный элемент
```

### 3.2 Светлая тема (опционально)

```css
/* При включении светлой темы используем Telegram-переменные */
/* --tg-theme-bg-color, --tg-theme-text-color и т.д. */
/* Все акцентные цвета остаются теми же */
```

### 3.3 Типографика

| Стиль | Шрифт | Размер | Вес | Межстрочный |
|---|---|---|---|---|
| H1 (заголовок экрана) | SF Pro / Roboto | 28pt | Bold 700 | 34pt |
| H2 (заголовок секции) | SF Pro / Roboto | 20pt | Semibold 600 | 25pt |
| H3 (заголовок карточки) | SF Pro / Roboto | 17pt | Semibold 600 | 22pt |
| Body (основной текст) | SF Pro / Roboto | 15pt | Regular 400 | 20pt |
| Body Bold | SF Pro / Roboto | 15pt | Semibold 600 | 20pt |

### 3.4 Моноширинный шрифт

Используется для:
- Показаний датчиков («22.5°C», «850 ppm»)
- IEEE-адресов устройств
- Статусов («ONLINE», «OFFLINE»)
- Времени в журнале («10:23:45»)

## 4. Компонентная библиотека

### 4.1 Button (Кнопка)

```
┌──────────────────────────────────┐
│         ТЕКСТ КНОПКИ             │  ← 44pt высота, радиус 12pt
└──────────────────────────────────┘

Состояния:
- Default:   bg = accent-blue, text = white
- Pressed:   bg = accent-blue * 0.7, scale(0.97)
- Disabled:  bg = bg-tertiary, text = text-tertiary
- Danger:    bg = accent-red, text = white
- Success:   bg = accent-green, text = #0D1117

Размеры:
- Large:  ширина = 100% контента, высота = 52pt
- Medium: ширина = по содержимому + 24pt, высота = 44pt
- Small:  ширина = по содержимому + 16pt, высота = 36pt
```

**Кнопка «Открыть ворота» (специальный стиль Danger/Confirm):**
Высота 64pt, полная ширина, анимированный фон при нажатии (pulse), требует удержания 0.5 сек.

### 4.2 Toggle (Переключатель вкл/выкл)

```
○────●  ВКЛ  (accent-green, точка справа)
●────○  ВЫКЛ (bg-tertiary, точка слева)

Размер: 51pt × 31pt
Точка: 27pt диаметр
Анимация: slide 200ms ease-out
```

### 4.3 Card (Карточка)

```
┌────────────────────────────────────┐
│  16px отступ                       │
```

## 5. Установка Mini App и первый запуск

### 5.1 Путь пользователя: как попасть в Mini App

```
Способ 1: Прямая ссылка
  Пользователь получает ссылку t.me/SmartEstateBot/app
  → Открывает в Telegram
  → Нажимает «Запустить»
  → Mini App открывается в WebView

Способ 2: Поиск в Telegram
  Поиск @SmartEstateBot
  → Профиль бота
  → Кнопка «Запустить» / Menu → «Открыть приложение»

Способ 3: QR-код (на корпусе Mac mini / в документации)
  Сканирует камерой → открывается бот → Mini App
```

### 15.4 Оптимистичный UI и откат

```
Пользователь нажимает Toggle → ВКЛ:

1. Toggle мгновенно переключается в ON (оптимистично)
2. Отправляется POST /api/devices/:id/on
3a. Успех → Toast «✅ Свет включён» (зелёный, 2 сек)
3b. Ошибка → Toggle возвращается в OFF + Toast «❌ Не удалось включить» (красный, 4 сек)
                + кнопка [ПОВТОРИТЬ] на карточке
```

```

(например, свет вкл через Zigbee-кнопку, а UI показывает выкл):

1. WebSocket присылает актуальное состояние
2. Toggle анимированно переключается (не мгновенно, а с анимацией)
3. Кратковременная подсветка карточки (accent-blue 0.1, 1 сек)
```

### C.9 Как датчики отображаются в интерфейсе

**Dashboard (климатическая карточка):**
- 🌡️ Температура + 💧 Влажность — основные значения
- CO₂ + PM2.5 — под основными, мелким шрифтом
- Статус: 🟢 Если все в норме / 🟡 Если повышен CO₂ / 🔴 Если протечка

**Dashboard (карточка энергии):**
- ⚡ Текущая мощность (Вт) — крупно
- 🔌 Потребление за сегодня (кВт·ч) — под ним

**Dashboard (карточка безопасности):**
- 🟢 Все датчики на связи / 🔴 Есть тревоги
- Счётчик: «0 тревог за 24 часа»

**Экран Устройств:**
- Каждый датчик — плитка с иконкой, названием, последним значением
- Для батарейных: индикатор заряда 🔋
- Для всех: качество связи 📶 (lqi)

**Экран Событий:**
- 🔵 Открытие двери / ворот
- 🟡 Сработал сценарий
- 🔴 Ошибка датчика / протечка
