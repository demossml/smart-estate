# Техническое задание для дизайнера
# Telegram Mini App «Умная Усадьба»

> **Дата:** 30 мая 2026  
> **Платформа:** Telegram Mini App (WebView внутри Telegram)  
> **Устройства:** Мобильные (iOS + Android), десктоп опционально  
> **Бэкенд:** готов, REST API + WebSocket, см. Приложение А  

---

## 1. Общее описание продукта

Умная Усадьба — Telegram Mini App для управления загородным домом и участком.  
Пользователь (владелец усадьбы) через телефон управляет:

- 💡 Освещением (внутри дома, на улице, в саду)
- 🌡️ Климатом (отопление, вентиляция, кондиционер)
- 💧 Поливом газона и растений
- 🚪 Воротами и калиткой
- 🔒 Охраной периметра
- 📊 Мониторингом датчиков (CO₂, температура, влажность, движение)

Приложение открывается прямо в Telegram, без установки.

## 2. Ключевые принципы дизайна

### 2.1 Минимализм и скорость
- Приложение для быстрого действия: открыл → нажал → закрыл
- Основные сценарии: «включить свет», «открыть ворота», «посмотреть температуру»
- Каждое ключевое действие — в 1-2 тапа

### 2.2 Тёмная тема по умолчанию
- **Тёмный фон** (как в Telegram)
- Неоновые акценты: голубой (информация), зелёный (вкл/ок), красный (тревога)
- Белый текст на тёмном — минимальная нагрузка на глаза

### 2.3 Адаптивность
- **Приоритет: iPhone SE (375pt) → iPhone 14 Pro Max (430pt)**
- Все элементы должны помещаться на одном экране без горизонтального скролла
- Крупные hit-зоны для пальцев: минимум 44pt × 44pt

### 2.4 Telegram-нативность
- Использовать цветовую схему Telegram (var(--tg-theme-*))
- Кнопки в стиле Telegram: скруглённые, акцентный цвет
- Нативный Header с названием приложения
- Haptic feedback на действия (вкл/выкл, открыть/закрыть)

## 3. Экраны приложения (6 основных экранов)

### 3.1 🏠 Главный экран (Dashboard)

**Назначение:** Быстрый обзор всего дома. Самый частый экран.

**Элементы:**
```
┌──────────────────────────┐
│ Умная Усадьба    [⚙️]   │  ← Header с настройками
├──────────────────────────┤
│ 🌡️ Гостиная   22.5°C    │  ← Карточка климата
│ ████████░░ 55% влажн.   │
├──────────────────────────┤
│ 💡 Освещение   🔵 3 вкл │  ← Карточка освещения
│ [ВКЛЮЧИТЬ ВЕСЬ СВЕТ]   │
├──────────────────────────┤
│ 🚪 Ворота       🔒       │  ← Карточка доступа
│ Последнее: 10 мин назад  │
├──────────────────────────┤
│ ⚡ Энергия   1.2 кВт·ч   │  ← Карточка энергии
│ за сегодня · 340 Вт      │
├──────────────────────────┤
│ 🔒 Охрана  ● НА СВЯЗИ   │  ← Статус безопасности
└──────────────────────────┘
```

**Данные:** `GET /api/status`, `GET /api/climate`, `GET /api/energy`

**Особенности:**
- Карточки раскладываются в 2 колонки на широких экранах
- Каждая карточка кликабельна → переход на соответствующий раздел
- Pull-to-refresh для обновления
- Живые данные через WebSocket (температура меняется сама)

---

### 3.2 💡 Управление устройствами (Devices)

**Назначение:** Включить/выключить любое устройство. Сгруппировано по комнатам.

**Элементы:**
```
┌──────────────────────────┐
│ ← Устройства    [🔍]    │
│ [Все] [🟢Вкл] [🔴Выкл] │  ← Фильтр-табы
├──────────────────────────┤
│ 🛋️ ГОСТИНАЯ    3 устр.  │  ← Группа по комнате
│ ┌────────────────────┐  │
│ │ 💡 Люстра    ● ВКЛ │  │  ← Устройство (слайд-переключатель)
│ │ 🔌 Торшер    ○ ВЫКЛ│  │
│ │ 🌡️ Термостат 22.5° │  │
│ └────────────────────┘  │
│ 🍳 КУХНЯ       2 устр.  │
│ 🌿 УЛИЦА       4 устр.  │
└──────────────────────────┘
```

**Данные:** `GET /api/devices`, `GET /api/rooms`, `POST /api/devices/:id/on`, `POST /api/devices/:id/off`

**Особенности:**
- Слайд-переключатель (toggle) для вкл/выкл — мгновенная реакция
- Лёгкая вибрация (haptic) при переключении
- Групповые действия: «Выключить всё в комнате» (свайп по названию комнаты)

### 3.3 🌡️ Климат-контроль

**Назначение:** Настройка температуры по комнатам.

**Элементы:**
```
┌──────────────────────────┐
│ ← Климат                │
├──────────────────────────┤
│ 🛋️ Гостиная             │
│ Текущая:   22.5°C       │
│             ██████░░░░   │
│ Заданная:  ─●──── 23°C  │  ← Слайдер уставки
│ Режим:     [🔥] [❄️] [🔄]│  ← Кнопки режима
│ Статус:    ✅ В норме    │
├──────────────────────────┤
│ 🛏️ Спальня              │
│ Текущая:   20.0°C       │
│ Заданная:  ──●─── 20°C  │
│ Статус:    🔥 Нагрев    │
└──────────────────────────┘
```

**Данные:** `GET /api/climate`, `PUT /api/climate/:device_ieee`

**Особенности:**
- Слайдер с плавной анимацией для выбора температуры
- Кнопки режимов с подсветкой активного
- Визуальная индикация: **зелёный** = в норме, **оранжевый** = нагрев, **синий** = охлаждение
- Обновление статуса в реальном времени

### 3.4 🚪 Ворота и доступ

**Назначение:** Управление воротами, калиткой, просмотр журнала.

**Элементы:**
```
┌──────────────────────────┐
│ ← Ворота и доступ       │
├──────────────────────────┤
│       🚪 ВЪЕЗДНЫЕ       │
│    ┌─────────────────┐  │
│    │   ЗАКРЫТЫ  🔒   │  │  ← Большая кнопка
│    │  Открыть        │  │
│    └─────────────────┘  │
├──────────────────────────┤
│       🚶 КАЛИТКА        │
│    ┌─────────────────┐  │
│    │   ЗАКРЫТА  🔒   │  │
│    └─────────────────┘  │
├──────────────────────────┤
│ 📋 Журнал доступа       │
│ 10:23  Открыты (API)    │
│ 09:45  Закрыты (сценар) │  ← Список последних событий
│ 08:00  Открыты (гость)  │
│ Вчера                    │
│ 22:00  Закрыты (авто)   │
└──────────────────────────┘
```

**Данные:** `GET /api/gates`, `POST /api/gates/:id/open`, `POST /api/gates/:id/close`, `GET /api/gates/access-log`

**Особенности:**
- Крупные кнопки «Открыть» / «Закрыть» (пол-экрана шириной) — на случай управления на ходу
- Подтверждение действия: «Точно открыть ворота?» (защита от случайных нажатий)
- Анимация открытия/закрытия: иконка ворот меняется
- Журнал с группировкой по дням

### 3.5 ⏱️ Сценарии (Automations)

**Назначение:** Просмотр и управление сценариями автоматизации.

**Элементы:**
```
┌──────────────────────────┐
│ ← Сценарии              │
├──────────────────────────┤
│ 🟢 Вентиляция по CO₂    │
│ CO₂ > 1000 → открыть     │
│              [ВЫКЛЮЧИТЬ] │
├──────────────────────────┤
│ 🟢 Свет при закате       │
│ Закат -30 мин → свет ON  │
│              [ВЫКЛЮЧИТЬ] │
├──────────────────────────┤
```

## 4. Дополнительные состояния

### 4.1 Пустые состояния (Empty States)
- «Нет устройств» → кнопка «Добавить устройство» с инструкцией
- «Нет событий» → «Всё спокойно» с мирной иконкой
- «Нет соединения» → иконка разрыва + кнопка «Повторить»

### 4.2 Загрузка
- Skeleton-экраны (серые мерцающие блоки) вместо пустого экрана
- Мгновенный оптимистичный UI: кнопка сразу показывает новое состояние, потом приходит подтверждение от API

### 4.3 Ошибки
- Ненавязчивый toast снизу: «Не удалось включить свет. Попробуйте ещё раз»
- Кнопка «Повторить» на карточке, если устройство не ответило
- Офлайн-режим: показывать закэшированные данные

## 6. Бэкенд (что уже готово)

Дизайнеру не нужно об этом думать, но для справки:

| Раздел | API Endpoints |
|---|---|
| Статус | `GET /api/status` |
| Устройства | `GET /api/devices`, `POST on/off` |
| Комнаты | `GET /api/rooms` |
| Климат | `GET /api/climate`, `PUT /api/climate/:id` |
| Ворота | `GET /api/gates`, `POST open/close`, `GET access-log` |
| Сценарии | `GET /api/scenarios`, `POST toggle` |
| Энергия | `GET /api/energy` |
| События | `GET /api/events`, `GET /api/audit` |
| WebSocket | `ws://host:8790/ws` — real-time обновления |
| Telegram auth | initData валидация на бэкенде |

**Все данные приходят в JSON. Аутентификация через Telegram initData.**

## Приложение А: Форматы данных API

### Устройство
```json
{
  "ieee_addr": "0x00124b...",
  "friendly_name": "Люстра гостиная",
  "type": "light",
  "status": "online",
  "room_name": "Гостиная",
  "room_icon": "🛋️",
  "latest_telemetry": [
    {"property": "state", "value": 1, "unit": "state"},
    {"property": "power", "value": 12.5, "unit": "W"}
  ]
}
```

### Климат
```json
{
  "device_ieee": "living_thermostat",
  "target_temp": 22.0,
  "current_temp": 22.5,
  "mode": "auto",
  "needs_heat": false,
  "needs_cool": false,
  "action": "idle"
}
```

### Событие
```json
{
  "type": "command",   // или "state_change", "error"
  "payload": "...",
  "ts": "2026-05-30T10:23:00Z"
}
```
