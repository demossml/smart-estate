## Дизайн-система (Design Tokens)

| Token | Value | Usage |
|-------|-------|-------|
| `--bg` | `#0A0C0B` | Фон страницы (почти чёрный, тёплый подтон) |
| `--glass-bg` | `rgba(20,26,22,0.6)` | Фон карточек (стекло) |
| `--glass-border` | `rgba(201,162,75,0.14)` | Кайма карточек |
| `--gold` | `#C9A24B` | Акцент, бренд, интерактивные элементы |
| `--green-live` | `#5CC98A` | Присутствие, "живое" состояние |
| `--green-dim` | `#4F8F68` | Присутствие muted |
| `--amber` | `#7A5C2E` | Приглушённый янтарь (вторичные элементы) |
| `--danger` | `#B23B34` | Тревога (протечка, окно открыто, батарея) |
| `--text-primary` | `#E9E4D8` | Основной текст |
| `--text-dim` | `#8B9088` | Приглушённый текст |
| `--text-muted` | `#5A5F58` | Ещё более приглушённый (метки) |

### Typography

| Назначение | Font | Weight |
|-----------|------|--------|
| Заголовки, числа | `Cormorant SC` (serif) | 500–700 |
| Акцентный текст | `Cormorant Garamond` (serif, italic) | 500–600 |

### 2. Tab: Сценарии (Scenarios)

**Header:**
```
СЦЕНАРИИ
IF-THEN правила движка автоматизации
```

**Список сценариев:**
```
┌──────────────────────────────────────────────────────────────┐
│  [ON]   ЕСЛИ Влажность > 70%  →  Включить вентилятор     🗑  │
├──────────────────────────────────────────────────────────────┤
│  [ON]   ЕСЛИ Температура < 18°C и время > 23:00 → Вкл. обогрев │
├──────────────────────────────────────────────────────────────┤
│  [OFF]  ЕСЛИ Никого нет 15 мин  →  Выключить всё         🗑  │
└──────────────────────────────────────────────────────────────┘
```

- Строка: переключатель + условие (золотые/янтарные ключевые слова) + действие
- Кнопка удалить (красная при наведении)
- **Добавление нового:** два input + кнопка "Добавить сценарий"

**8 предустановленных сценариев:**
1. ✅ Влажность > 70% → Вентилятор
2. ✅ Температура < 18°C + ночь → Обогрев
3. ✅ Окно открыто + кондиционер → Выкл кондиционер
4. ✅ Протечка → Уведомление + перекрыть воду
5. ✅ Движение + освещённость < 100 lux → Свет
6. ❌ Никого нет 15 мин → Выкл всё
7. ✅ Рассвет → Открыть шторы
8. ✅ Закат → Включить ночник

### 4. Модальные окна

#### Add Device Modal (2 шага)

**Шаг 1 — Выбор типа:**
- Grid 2×5 из всех типов устройств с иконками
- Выбор → кнопка "Далее"

**Шаг 2 — Настройка:**
- Поле "Имя устройства"
- Переключатель "Существующая комната" / "Новая комната"
- Если существующая: select
- Если новая: input + иконка (5 вариантов: hallway/living/bedroom/kitchen/yard)
- Кнопка "Добавить устройство"
- Анимация поиска (Loader spin + "Поиск устройства через MQTT…")

#### Add Room Modal

- Поле "Название"
- Иконки (5 вариантов)
- Кнопка "Создать комнату"

### 5. Визуальные компоненты

#### Glass-карточки
```css
border-radius: 16px;
background: linear-gradient(165deg, rgba(24,30,25,0.65), rgba(14,18,15,0.65));
backdrop-filter: blur(14px);
border: 1px solid rgba(201,162,75,0.14);
```

#### Переключатели (Switch)
- 30×17px, скруглённые
- OFF: серый фон `rgba(255,255,255,0.08)`
- ON: зелёный фон `#3E7A56`
- Knob: белый, 13px, сдвиг на 13px в ON

#### Slider (ползунок яркости)
- Трек: золотой градиент
- Thumb: золотой 12px, с обводкой фона

#### Анимации
- **Разворачивание карточки:** max-height 0 → 900px, 380ms cubic-bezier
- **Присутствие:** зелёный glow на иконке комнаты
- **Поиск устройства:** spin иконки Loader
- **Bottom nav:** золотой цвет для активной вкладки

#### Статус-бейджи
- ✅ OK: `#7FE0A8` (зелёный)
- ⚠️ Alert: `#D9695F` (красный, с фоном `rgba(178,59,52,0.14)`)
- 📶 Сигнал: приглушённый серый `#5A5F58`
- 🔋 Батарея: `<10%` красный, `<20%` жёлтый, `>20%` зелёный

## План интеграции в реальное приложение

1. **Выделить дизайн-токены** в CSS-переменные
2. **Разбить компоненты** по файлам: `RoomTileV2.tsx` → `DeviceCard.tsx`, `ScenarioList.tsx`, `EnergyPanel.tsx`
3. **Подключить real API** вместо INITIAL_* данных
4. **Добавить WebSocket** для live-обновлений
5. **Интегрировать настройки** (чувствительность, тайм-аут) → MQTT publish
6. **Добавить гистограмму** из `/api/telemetry`
7. **Добавить энергомониторинг** из `/api/energy`
